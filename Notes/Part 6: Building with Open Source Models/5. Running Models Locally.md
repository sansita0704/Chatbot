# Running Models Locally

- With the current code, we are calling the Hugging Face API behind the scenes, just like we call the OpenAI API.
- This means our prompts and responses travel over the network.
- If we want complete privacy, we can run the model locally on our own machine.
- To do this, we use **Ollama**.

## Ollama

> https://ollama.com

- Ollama is a free, cross-platform tool that makes it incredibly easy to run language models locally.
- It is like Docker, but for LLMs.
- Ollama has its own model registry.
- However, it can also run compatible Hugging Face models locally.

### Download Ollama

> https://ollama.com/download

Download and install Ollama.

Then run:

```bash
ollama
```

in a terminal window to verify that it is installed correctly.

### Install a Model

Run:

```bash
ollama run llama3.1
```

- Since this model does not exist on our machine yet, Ollama will automatically download it from its registry.
- After installation, Ollama opens an interactive shell where we can chat with the model.

### List Installed Models

```bash
ollama list
```

Displays all models currently stored on the machine.

### View Running Models

```bash
ollama ps
```

Displays all models that are currently running.

### Note

- These models can take up a significant amount of disk space.
- Once you are done using them, remove them to free up storage.

### Remove a Model

```bash
ollama rm <model-name>
```

`rm` stands for **remove**.

Example:

```bash
ollama rm llama3.1
```
