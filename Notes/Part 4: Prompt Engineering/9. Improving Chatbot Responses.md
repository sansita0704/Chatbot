# Improve Chatbot Responses

## Move the prompt into a separate file

- Instead of hardcoding the system prompt inside the service, create a dedicated directory for prompts.

Create `server/prompts/chatbot.txt`:

```text
You are a customer support agent for a theme park named WonderWorld.

Here's some key information about the park:

{{parkInfo}}

Only answer questions related to WonderWorld.
Always answer in a cheerful tone and avoid making up information.
```

### Why use `{{parkInfo}}`?

- `{{parkInfo}}` is a placeholder that will later be replaced with the contents of `WonderWorld.md`.

We intentionally **don't copy and paste** the park information directly into the prompt because these are two separate concerns:

- **The prompt** defines how the AI should behave.
    - It contains instructions like tone, scope, and response style.
    - Over time, we may refine these instructions to improve response quality.

- **The park information** is the knowledge the AI should use.
    - It is separate content that may change independently.
    - Keeping it in a Markdown file makes it much easier to update without touching the prompt.

Separating these concerns makes the code cleaner and more maintainable.

### Import the prompt template

Import the text prompt into the chat service.

```ts
import template from "../prompts/chatbot.txt";
```

Text files can be imported directly.

### Read the Markdown file

- Unlike `.txt` imports, the Markdown file should be read using Node's File System module.

Import the required modules:

```ts
import fs from "fs";
import path from "path";
```

Read the file synchronously:

```ts
const parkInfo = fs.readFileSync(
    path.join(__dirname, "..", "prompts", "WonderWorld.md"),
    "utf-8",
);
```

### `path.join()` arguments

Each argument contributes to building the final file path:

- `__dirname` → Current directory
- `".."` → Move one directory up

Using `path.join()` ensures paths work correctly across different operating systems.

### `readFileSync()` arguments

`readFileSync()` accepts two arguments:

1. **Path** to the file
2. **Encoding**

```ts
fs.readFileSync(path, "utf-8");
```

### Replace the placeholder

Replace `{{parkInfo}}` with the contents of the Markdown file.

```ts
const instructions = template.replace("{{parkInfo}}", parkInfo);
```

- Now `instructions` contains the complete system prompt that will be sent to the LLM.
- This replacement happens **once when the module is first loaded**, and the resulting prompt is reused for every API request instead of being rebuilt each time.

### Create the system message

Create a system message using the generated instructions.

```ts
const systemMessage = {
    role: "system",
    content: instructions,
};
```

### Update the API request

Pass the system message before the chat history.

```ts
const response = await client.chat.completions.create({
    model: "llama-3.1-8b-instant",
    messages: [systemMessage, ...chatHistory],
    temperature: 0.2,
    max_tokens: 300,
});
```
