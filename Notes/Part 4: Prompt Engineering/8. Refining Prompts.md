# Building and Refining a Prompt

## Goal

Build and improve a prompt for a **theme park customer support assistant**.

The process involves:

1. Starting with a simple prompt.
2. Testing the model.
3. Refining the prompt based on its responses.

### Step 1: Test Without Any Context

Ask the model:

> What time does the park close?

Response:

> Could you please specify which park you are referring to?

### Step 2: Give the Model Context

Provide system instructions such as:

```text
You are a customer support agent for a theme park named WonderWorld.

Park opening hours:
- Monday–Thursday: 9 AM – 5 PM
- Friday–Sunday: 9 AM – 9 PM
```

Now ask again:

> What time does the park close?

Response:

> WonderWorld's closing times are:
>
> - Monday–Thursday: 5 PM
> - Friday–Sunday: 9 PM

This is an improvement because the model now has the necessary context.

### Step 3: Handle Questions About "Today"

User asks:

> What time does the park close today?

### Problem

LLMs **do not know the current date or day**.

Response:

> Could you please tell me what day of the week it is today?

This is technically correct but not a great user experience.

### Step 4: Provide Today's Date

Include the current date in the prompt.

Example:

```text
Today is Thursday, July 17.
```

> Note:
>
> In the Playground, the date is hardcoded.  
> In a real application, this value should be **inserted dynamically**.

Now ask again:

> What time does the park close today?

Response:

> Today is Thursday, July 17. WonderWorld closes at 5 PM today.

The answer is now correct.

### Step 5: Improve the Response

- The response unnecessarily repeats today's date.
- To make it sound more natural, refine the prompt with an additional instruction:

```text
Do not repeat today's date in your responses.
```

Ask again:

> What time does the park close today?

Response:

> WonderWorld closes at 5 PM today.

This is a more natural response.

### Step 6: Prevent Off-Topic Questions

Ask an unrelated question:

> What is the capital of France?

- Without restrictions, the model answers the question, even though it has nothing to do with the theme park.
- This is undesirable for a customer support chatbot.

### Step 7: Restrict the Model's Scope

Add another instruction:

```text
Only answer questions related to WonderWorld.
```

Ask again:

> What is the capital of France?

Response:

> I'm here to help with any questions you have about WonderWorld.

This keeps the chatbot focused on its intended purpose.

### Step 8: Test Edge Cases

Enter gibberish or unclear text.

Example:

```text
asdhjkasd123!!!
```

Response:

> It looks like your message may have been mistyped or is unclear. If you have any questions about WonderWorld, feel free to ask and I'll be happy to help.

This is a good default response for invalid or unclear input.

### Prompt Engineering is an Iterative Process

- Building good prompts is similar to writing code.
- It is **not** something you usually get right on the first attempt.
