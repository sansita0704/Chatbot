# Providing Context in Prompts

## Why Context Matters

- A basic chatbot can answer questions because it is essentially a wrapper around a general-purpose language model.

However, without additional context, it:

- Doesn't know about our business, customers and products.
- Can only provide general-purpose answers.

The real value comes from **providing the model with relevant context**.

By doing so, we transform a generic assistant into one that can:

- Answer business-specific questions
- Provide accurate information
- Respond in the desired tone
- Help users find information quickly

# Ways to Provide Context

There are several simple techniques for supplying context to the model.

## 1. Assign a Role

- One of the easiest ways to guide the model is to assign it a role.
- The model will adjust its responses based on that role.

### Example 1

```text
You're a senior backend engineer.
Explain the code below to a junior developer.
```

### Example 2

```text
You're a customer support agent.
Respond to this customer in a polite and empathetic tone.
```

### Effect of Assigning a Role

Assigning a role does **not** give the model new capabilities.

Instead, it changes:

- Tone
- Structure
- Focus
- Style of communication

## 2. Supply Background Information

- Provide the model with relevant facts, rules, or information needed to complete the task.

### Example

```text
You're a customer support assistant.
Here is our refund policy.
Now answer this customer's question.
```

- Background information helps the model generate responses that are accurate and aligned with our business or application.

## 3. Specify the Target Audience

- Tell the model who the response is intended for.

This helps control:

- Complexity
- Vocabulary
- Explanation style

### Examples

```text
Explain this topic to a non-technical user.
```

```text
Write this summary for a high school student.
```

The model adjusts its language and level of detail accordingly.

## 4. Specify the Desired Tone

- Tone plays an important role, especially in customer-facing applications.
- We can explicitly instruct the model how it should sound.

### Example 1

```text
Respond in a friendly, conversational tone.
Avoid sounding formal or robotic.
```

### Example 2

```text
Use professional and empathetic language like a calm support representative helping a frustrated customer.
```

These instructions make interactions feel more natural and human.

## 5. Providing Reference Material

- Sometimes the model needs actual content to work with.

Examples include:

- Product descriptions
- Customer reviews
- Emails
- Transcripts
- Documentation

Instead of mixing the instructions with the reference material, it's best to separate them visually.

Common separators include:

- Triple dashes (---)
- Triple quotes (`"""`)
- XML-style tags (<input>...</input>)

There is **no strict rule** about which separator to use.

The important part is to clearly distinguish:

- Instructions
- Reference material

## Key Idea

- Providing context does **not** mean writing unnecessarily long prompts or repeating instructions.

It simply means giving the model the information it needs to produce responses that are:

- Accurate
- Relevant
- Useful

Adding the right context transforms generic AI responses into practical application functionality.
