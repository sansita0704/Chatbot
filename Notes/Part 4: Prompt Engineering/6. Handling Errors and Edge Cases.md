# Handling Errors

- Writing prompts is similar to writing functions.
- When testing a function, we don't only test the **happy path** (valid input), we also test how it behaves with **invalid or unexpected input**.
- Prompt engineering follows the same principle.

Even if a prompt works well with clean input, we should verify how it handles:

- Vague input
- Empty input
- Invalid input
- Unexpected input

The goal is to make prompts **more reliable and robust**.

## Strategy 1: Return an Error

- One approach is to instruct the model to return a predefined error when the input is missing or invalid.

### Example

```text
Summarize this product review.
If the input is empty or not a valid review, respond with:

{ error: "Invalid input" }
```

### Handle Very Short Input

We can also define rules for inputs that are too short.

```text
If the input contains fewer than five words, return:

{ error: "Too short to summarize" }
```

The exact validation rules depend on the application being built.

## Strategy 2: Ask a Clarifying Question

- Instead of returning an error, chatbots can ask the user for more information.
- This approach is appropriate for **conversational applications**.

### Example

```text
If the user's request is too vague or lacks detail, ask a clarifying question instead of guessing.
```

This helps prevent the model from making incorrect assumptions.

## Always Test Your Prompts

- Just like software, prompts should be thoroughly tested.
- Test them with different kinds of input to identify weaknesses.

Recommended test cases include:

- Empty string
- Only whitespace
- One or two random words
- Gibberish or meaningless text
- Extremely long input
- Content missing important details

Observe how the model responds in each case.

### Refine the Prompt

If testing reveals that the model:

- Breaks
- Produces poor-quality output

Then revise and improve the prompt.
