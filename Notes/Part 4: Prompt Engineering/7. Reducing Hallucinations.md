# What are Hallucinations?

- **Hallucinations** occur when an LLM generates information that is false or made up.
- The model is **not intentionally lying**; it is simply predicting the most likely next words based on patterns in its training data.

## Why Do Hallucinations Happen?

LLMs **do not know facts** or look up information in a database.

Instead, they:

- Predict what text should come next.
- Rely on patterns learned during training.

Hallucinations are more likely when:

- The prompt is **vague**.
- The question is **open-ended**.
- The prompt **doesn't provide enough information**.

In these situations, the model fills in the missing information with what it thinks sounds most likely.

# How to Reduce Hallucinations

## 1. Provide Facts in the Prompt (Grounding)

If you want accurate answers, **give the model the necessary information** instead of expecting it to know it.

### Example

Customer asks:

> Can I cancel my ticket for tomorrow?

Instead of expecting the model to know your company's refund policy, provide it in the prompt:

```
Here's our refund policy:
"..."

Now answer this customer's question:
Can I cancel my ticket for tomorrow?
```

- This technique is called **grounding**.
- **Grounding:** Anchoring the model's response to real, provided information rather than letting it guess.

## 2. Tell the Model What to Do When It Doesn't Know

- Sometimes the answer isn't available or the input is unclear.
- Instead of allowing the model to guess, explicitly instruct it to admit uncertainty.

Example instruction:

```
If you're unsure or the answer isn't available, say:
"Sorry, I don't have that information."

Do not guess or make up a response.
```

- This teaches the model that **not answering is better than inventing information**.

## 3. Limit the Model's Scope

Hallucinations often occur when the model answers questions outside its intended purpose.

### Example

- Suppose you build a chatbot for a **theme park**.

Appropriate questions:

- Rides
- Tickets
- Opening hours
- Park policies

Unrelated questions:

- Write me a Python script.
- What's the capital of Brazil?

Without instructions, the model may attempt to answer everything.

### Prevent this by setting boundaries

Example prompt:

```
You are a support assistant for a theme park.
Only answer questions related to the park, including rides, tickets, hours, and policies.

If the question is unrelated, respond with:

"I'm here to help with park-related questions. Let me know if there's anything you would like to know about your visit."
```

Setting clear boundaries helps the model:

- Stay focused.
- Avoid off-topic responses.
- Reduce hallucinations.

## 4. Don't Over-Trust the Model

- Even with good prompts, hallucinations can still happen, especially with **general-purpose LLMs**.
- Always treat AI outputs as something that may need verification.
    - Always sanitize or validate outputs
    - Avoid using AI for **critical decisions** without human review.
    - Log model outputs.
    - Monitor for unusual or incorrect behavior.

### Validate Model Outputs

Always sanitize or validate outputs before using them.

For example:

- If the model returns a **JSON object**, validate it using a data validation library like **Zod**.
- This is similar to validating API inputs.
