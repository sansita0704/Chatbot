# Improving Copy Behavior

- When selecting and copying chatbot messages, browsers often copy extra whitespace and invisible formatting from the HTML structure.
- As a result, pasted text may contain unnecessary blank lines or inconsistent formatting.
- We can customize the copy behavior so that only the selected plain text is copied.

## Create a Copy Handler

```tsx
const onCopyMessage = (e: React.ClipboardEvent): void => {
    const selection = window.getSelection()?.toString().trim();

    if (selection) {
        e.preventDefault();
        e.clipboardData.setData("text/plain", selection);
    }
};
```

### How It Works

This function handles the `onCopy` event, so it takes the event object as its argument.

### Step 1: Get the Selected Text

- `window.getSelection()` returns the currently selected text.
- `toString()` converts it into a string.
- `trim()` removes leading and trailing whitespace.

### Step 2: Stop the Browser's Default Copy Behavior

- Normally, the browser copies both the text and the underlying HTML formatting.
- Preventing the default behavior allows us to define exactly what gets copied.

### Step 3: Copy Only Plain Text

- `"text/plain"`: format
- `selection`: cleaned, trimmed text.

As a result, the pasted content no longer contains unnecessary formatting or extra line breaks.

## Attach the Event Handler

The messages are rendered inside a `<div>`, so attach the `onCopy` event to that element.

```tsx
<div
    key={index}
    onCopy={onCopyMessage}
    className={`px-3 py-1 rounded-xl ${
        message.role === "user" ?
            "bg-blue-600 text-white self-end"
        :   "bg-gray-100 text-black self-start"
    }`}
>
    <ReactMarkdown>{message.content}</ReactMarkdown>
</div>
```

Now, whenever the user copies a message, only the cleaned plain text is placed into the clipboard.
