# Handling Form Submission

- To handle form submission, we will use a library called **React Hook Form**.

## Install React Hook Form

In the client directory:

```bash
bun add react-hook-form
```

Import the hook:

```tsx
import { useForm } from "react-hook-form";
```

### Define Form Data Type

Create a type representing the form fields:

```tsx
type FormData = {
    prompt: string;
};
```

In this form, there is one field:

- `prompt` → string

## Initialize React Hook Form

Call `useForm()` and provide the form data type as a generic:

```tsx
const { register, handleSubmit } = useForm<FormData>();
```

### Functions Used

- `register` → registers form fields
- `handleSubmit` → handles form submission

## Register Fields

- Registering means adding the field or that HTML element to React Hook Form's internal list of managed fields.
- This connects the HTML element to the hook so it can manage its value, track state changes, and apply validation rules.

Register the `textarea` with React Hook Form:

```tsx
<textarea
    {...register("prompt", {
        required: true, // this field is required
    })}
/>
```

### Arguments

1. `name` property: name of the field → "prompt"
2. `options` object: to define validation rules → optional

### Notes

- `"prompt"` matches the property in `FormData`.
- TypeScript provides autocomplete based on `FormData`.
- `register()` returns an object containing props that must be added to the textarea element.
- Spread operator (`...`) adds all those props to the textarea.

### Convert Div to Form

- Replace `<div>` with `<form>`.
- Add form submission handling:

```tsx
<form onSubmit={handleSubmit((data) => {
  console.log(data);
})}>
```

### How `handleSubmit` Works

```tsx
handleSubmit((data) => {
    console.log(data);
});
```

- It takes a function as an argument that executes only if form validation succeeds.
- `data` is automatically typed as `FormData`.

Example output:

```js
{
    prompt: "test";
}
```

## `handleSubmit()`

> https://react-hook-form.com/docs/useform/handlesubmit

- `handleSubmit` is a **Higher-Order Function (HOF)**.
- It acts as a gatekeeper for form submissions.
- It does not handle data directly.
- Instead, it returns a wrapper function (an event handler) that intercepts the native HTML submit event, runs validations, and conditionally executes our custom callback.

### Execution Timeline (The Lifecycle)

The process happens in three distinct phases:

### Phase 1: Setup (On Component Render)

- **What happens:** `handleSubmit(myCallback)` is executed immediately when the React component renders.
- **Output:** It returns a wrapper/gatekeeper function which is attached to `<form onSubmit={...}>`.

### Phase 2: Trigger (On User Click / Form Submit)

- **What happens:** The user submits the form. The **returned wrapper function** fires first.
- **Actions taken automatically:**

1. Executes `e.preventDefault()` to stop browser page reloads.
2. Triggers internal validation rules registered on all input fields.

### Phase 3: Resolution (Conditional Execution)

- **If Form is Valid:** The wrapper function invokes our custom callback (e.g., `myCallback(data)`) and passes it the clean, validated form data object.
- **If Form is Invalid:** Our callback is **ignored**. The wrapper populates the `errors` object and triggers a component re-render to display validation errors.

### Reset Form After Submission

```tsx
const { register, handleSubmit, reset } = useForm<FormData>();
```

Extract submission logic into a separate function.

Create submit handler:

```tsx
function onSubmit(data: FormData) {
    console.log(data);
    reset();
}
```

Use it:

```tsx
<form onSubmit={handleSubmit(onSubmit)}>
```

### Notes

Passing:

```tsx
handleSubmit(onSubmit);
```

means passing a **function reference**.

Not:

```tsx
handleSubmit(onSubmit());
```

which would immediately invoke the function.

## Submit Form on Enter Key

Add `onKeyDown` to the textarea:

```tsx
onKeyDown={(e) => {
  if (e.key === "Enter" && !e.shiftKey) {
    handleSubmit(onSubmit)();
  }
}}
```

### Logic

Submit only when:

- Enter is pressed
- Shift key is not pressed

Reason:

- `Shift + Enter` should insert a new line.
- `Enter` alone should submit the form.

### Why `handleSubmit(onSubmit)()`?

`handleSubmit(onSubmit)` returns a function.

Therefore:

```tsx
handleSubmit(onSubmit)();
```

means:

1. Create submission function.
2. Immediately execute it.

This differs from:

```tsx
onSubmit={handleSubmit(onSubmit)}
```

where only a function reference is passed.

### Prevent Default Enter Behavior

Pressing Enter inside a textarea normally inserts a new line.

Prevent that:

```tsx
if (e.key === "Enter" && !e.shiftKey) {
    e.preventDefault();
    handleSubmit(onSubmit)();
}
```

### Move Key Handling Logic to Separate Function

Instead of inline code:

```tsx
onKeyDown={(e) => { ... }}
```

Create:

```tsx
function onKeyDown(e: KeyboardEvent<HTMLFormElement>) {
    if (e.key === "Enter" && !e.shiftKey) {
        e.preventDefault();
        handleSubmit(onSubmit)();
    }
}
```

Use:

```tsx
onKeyDown = { onKeyDown };
```

### Type Annotation for Event

- When moving the handler outside JSX, TypeScript cannot infer the type automatically.
- Explicitly provide `KeyboardEvent<HTMLFormElement>`.

Example:

```tsx
function onKeyDown(e: KeyboardEvent<HTMLFormElement>) {
  ...
}
```

### Note

Inside JSX:

```tsx
onKeyDown={(e) => {}}
```

Type is inferred automatically.

Outside JSX:

```tsx
function onKeyDown(e) {}
```

Type must be specified manually.

## Access Form State

Get form state from `useForm()`:

```tsx
const { register, handleSubmit, reset, formState } = useForm<FormData>();
```

### Disable Submit Button Until Form Is Valid

```tsx
<Button
    disabled={!formState.isValid}
    className="cursor-pointer rounded-full w-9 h-9"
></Button>
```

### Behavior

- Empty input → button disabled
- Valid input → button enabled

### Problem with `required`

Using only:

```tsx
required: true;
```

allows `"     "` because whitespace is still considered non-empty text.

- So, it passes validation even though it contains no meaningful content.

### Add Custom Validation

Use `validate`:

```tsx
register("prompt", {
    required: true,
    validate: (data) => data.trim().length > 0,
});
```

## Code Structure

```tsx
import { useForm } from "react-hook-form";
import type { KeyboardEvent } from "react";

type FormData = {
    prompt: string;
};

const ChatBot = () => {
    const { register, handleSubmit, reset, formState } = useForm<FormData>();

    const onSubmit = (data: FormData) => {
        reset();
        console.log(data);
    };

    const onKeyDown = (e: KeyboardEvent<HTMLFormElement>) => {
        if (e.key === "Enter" && !e.shiftKey) {
            e.preventDefault();
            handleSubmit(onSubmit)();
        }
    };

    return (
        <form
            onSubmit={handleSubmit(onSubmit)}
            onKeyDown={onKeyDown}
            className="..."
        >
            <textarea
                {...register("prompt", {
                    required: true,
                    validate: (data) => data.trim().length > 0,
                })}
                className="..."
                placeholder="Ask anything"
                maxLength={1000}
            ></textarea>
            <Button disabled={!formState.isValid} className="..."></Button>
        </form>
    );
};
```
