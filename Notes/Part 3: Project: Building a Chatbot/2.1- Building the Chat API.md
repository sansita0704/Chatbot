# Building a Chat API Endpoint

Goal: Create an API endpoint that receives a user's message and returns an AI-generated response.

## Install Dependency

Inside the `server` directory:

```bash
bun add @google/genai
```

## Setup Gemini

### 1. Import GoogleGenAI

```ts
import { GoogleGenAI } from "@google/genai";
```

### 2. Create a Gemini Client

Create a new Gemini client using your API key:

```ts
const client = new GoogleGenAI({
    apiKey: process.env.GEMINI_API_KEY,
});
```

## Create Chat Endpoint

Add the following route in `server/index.ts`:

```ts
app.post("/api/chat", async (req: Request, res: Response) => {
    // Get user's prompt
    const { prompt } = req.body;

    // Send prompt to Gemini
    const response = await client.models.generateContent({
        model: "gemini-2.5-flash",
        contents: prompt,
        config: {
            temperature: 0.2,
            maxOutputTokens: 100,
        },
    });

    res.json({ message: response.text });
});
```

## Understanding the Endpoint

### Why Use POST Instead of GET?

- We are not simply retrieving information.
- The user is **sending data (the prompt)** to the server, so we use an HTTP **POST** request.

### GET vs POST

Every HTTP request follows the same cycle:

```text
Client → Request → Server
Client ← Response ← Server
```

The client sends a request and the server must send back a response.

### GET Request

- Used to retrieve data.

Characteristics:

- Read-only
- Should not modify server state
- Used for fetching information

### POST Request

- Used to send data to the server.

Characteristics:

- Creates or updates resources
- Usually contains data in the request body

### Getting the User's Prompt

- Assume the request body contains a property named `prompt`.

```ts
const { prompt } = req.body;
```

Equivalent to:

```ts
const prompt = req.body.prompt;
```

### Sending the Prompt to Gemini

Use:

```ts
client.models.generateContent();
```

- Use `config` property to set `temperature` and `maxOutputTokens`.

### Temperature

```ts
temperature: 0.2 - 0.4;
```

- Chatbots usually need accuracy and consistency more than creativity.

### Max Output Tokens

```ts
maxOutputTokens: 100;
```

Benefits:

- Faster responses
- Lower cost
- Better user experience in chat applications

Without a limit, responses may become unnecessarily long.

### Returning the Response

Send the AI response back to the client:

```ts
res.json({
    message: response.text,
});
```

Response:

```json
{
    "message": "Hello! How can I help you?"
}
```

### Why `req.body` Doesn't Work Initially?

We are doing:

```ts
const { prompt } = req.body;
```

- However, by default Express does not automatically parse JSON request bodies.
- Without additional configuration `req.body` will be `undefined`.

### Parsing

Parsing means:

> Converting data from one format into another format that a program can understand and work with easily.

In Express, parsing converts a **JSON string into a JavaScript object**.

### Before Parsing

When a request arrives, the body is raw text:

```json
"{\"name\":\"Sansita\",\"age\":20}"
```

- This is just a string.
- Trying `body.name` will not work because strings do not have a `name` property.

### After Parsing

The JSON string becomes a JS object:

```ts
{
    name: "Sansita",
    age: 20
}
```

Now:

```ts
body.name; // "Sansita"
body.age; // 20
```

works correctly.

### Add a Middleware Function

To automatically parse incoming JSON requests:

```ts
app.use(express.json());
```

- This middleware runs before route handlers and parses JSON request bodies.
- After adding it `req.body` contains the parsed JS object.

### What is Middleware?

Middleware is a function that executes between:

```text
Incoming Request
        ↓
Middleware
        ↓
Route Handler
        ↓
Response
```

Example:

```ts
app.use(express.json());
```

Common uses of middleware:

- Parsing request data
- Authentication
- Authorization
- Logging
- Security checks

An Express application can have multiple middleware functions.

## Request Flow

When a request is received:

1. Request reaches Express.
2. `express.json()` middleware executes.
3. Parsed data is stored in `req.body`.

4. Route handler executes.
5. Gemini generates a response.
6. Response is sent back to the client.
