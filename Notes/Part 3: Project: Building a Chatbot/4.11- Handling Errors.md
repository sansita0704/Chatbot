# Handling Errors

Declare a state variable to store an error message.

```tsx
const [error, setError] = useState("");
```

- Whenever an error occurs, we'll update this state so it can be displayed to the user.

### Wrap the Submit Handler in a `try...catch`

- Since the operation inside `onSubmit()` can fail, wrap the entire function body in a `try...catch` block.

### Log the Error

Inside the `catch` block, log the error.

```tsx
console.error(error);
```

- In a real application, we should log errors using a logging service like **Sentry**.
- This allows us to monitor logs and quickly identify errors in our application.

For this project, we'll simply log the error to the browser console.

### Show an Error Message

```tsx
setError("Something went wrong, try again!");
```

Regardless of whether the request succeeds or fails, the typing indicator should disappear.

Use a `finally` block:

```tsx
finally {
    setIsBotTyping(false);
}
```

## Final Code

```tsx
const onSubmit = async ({ prompt }: FormData) => {
    try {
        setMessages((prev) => [
            ...prev,
            {
                content: prompt,
                role: "user",
            },
        ]);

        setIsBotTyping(true);

        reset({ prompt: "" });

        const { data } = await axios.post<ChatResponse>("/api/chat", {
            prompt,
            conversationId: conversationId.current,
        });

        setMessages((prev) => [
            ...prev,
            {
                content: data.message,
                role: "bot",
            },
        ]);
    } catch (error) {
        console.error(error);
        setError("Something went wrong, try again!");
    } finally {
        setIsBotTyping(false);
    }
};
```

### Render the Error Message

Display the error only when the `error` state contains a value.

```tsx
{
    error && <p className="text-red-500">{error}</p>;
}
```

### Reset Previous Errors

- Suppose one request fails.
- The error message is displayed.
- If the next request succeeds, the previous error should disappear.

Therefore, clear the error at the beginning of every submission.

```tsx
try {
    setError("");

    setMessages((prev) => [
        ...prev,
        {
            content: prompt,
            role: "user",
        },
    ]);

    setIsBotTyping(true);

    ...
}
```

This ensures only the latest request determines whether an error is displayed.
