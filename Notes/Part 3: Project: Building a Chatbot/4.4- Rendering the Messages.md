# Rendering Messages

Declare a state variable to store all chat messages.

```tsx
const [messages, setMessages] = useState<string[]>([]);
```

## Updating the messages array

When the user submits the form:

### 1. Add the user's prompt

```tsx
setMessages([...messages, prompt]);
```

### How it works

- `...messages` creates a copy of the existing messages array.
- `prompt` is appended to the end of that copied array.

### 2. Add the AI's response

After receiving the response from the server:

```tsx
setMessages([...messages, data.message]);
```

### Annotating the Response Object

Instead of letting TypeScript infer the response type, define it explicitly.

```tsx
type ChatResponse = {
    message: string;
};
```

Now pass it as a generic to `axios.post()`:

```tsx
const { data } = await axios.post<ChatResponse>("/api/chat", {
    prompt,
    conversationId: conversationId.current,
});
```

- With this, we get intellisense while typing `data.`.

### Rendering Messages

Render every message stored inside the array.

```tsx
<div>
    {messages.map((message, index) => (
        <p key={index}>{message}</p>
    ))}
</div>
```

However, this does **not** work correctly.

### Why Does the AI Response Replace the User Message?

Consider this `onSubmit()` function:

```tsx
const onSubmit = async ({ prompt }: FormData) => {
    setMessages([...messages, prompt]);

    reset();

    const { data } = await axios.post<ChatResponse>("/api/chat", {
        prompt,
        conversationId: conversationId.current,
    });

    setMessages([...messages, data.message]);
};
```

- There are **two** calls to `setMessages()`.
- Both calls use the same value of `messages`.

At the time `onSubmit()` begins executing:

```tsx
messages = [];
```

### First update

Creates:

```tsx
["hi"];
```

### Second update

Still uses the original empty array:

```tsx
["bot"];
```

- Notice that React has **not** updated `messages` yet.
- Both updates were created from the same stale copy: `[]`.

The second update overwrites the first one, causing the user's prompt to disappear.

### Solution

- Instead of using the current value of `messages`, tell React to provide the latest state.

```tsx
setMessages((prev) => [...prev, prompt]);
```

Now every update builds upon the latest state.
