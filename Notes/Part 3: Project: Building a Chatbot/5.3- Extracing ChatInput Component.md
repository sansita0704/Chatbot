# Chat Input Component

## Create the `ChatInput` Component

Create a new file `chat/ChatInput.tsx` and paste the entire `form` element:

```tsx
const ChatInput = () => {
    return (
        <form
            onSubmit={handleSubmit(onSubmit)}
            onKeyDown={onKeyDown}
            className="flex flex-col gap-2 items-end border-2 p-4 rounded-2xl"
        >
            <textarea
                {...register("prompt", {
                    required: true,
                    validate: (data) => data.trim().length > 0,
                })}
                autoFocus
                className="w-full border-0 outline-0 resize-none text-sm"
                placeholder="Ask anything"
                maxLength={1000}
            ></textarea>
            <Button
                disabled={!formState.isValid}
                className="cursor-pointer rounded-full w-9 h-9"
            >
                <FaArrowUp />
            </Button>
        </form>
    );
};

export default ChatInput;
```

### Replace the Form in ChatBot

In `chatbot.tsx`:

```tsx
<ChatInput />
```

Now `ChatBot` renders the new component instead of containing the form itself.

### Add Required Imports

```jsx
import { FaArrowUp } from "react-icons/fa";
import { Button } from "../ui/button";
```

### Move `useForm` into `ChatInput`

Previously, `ChatBot` contained:

```tsx
const { register, handleSubmit, reset, formState } = useForm<ChatFormData>();
```

Since this logic belongs to the form itself, move it into `ChatInput`.

Import:

```tsx
import { useForm } from "react-hook-form";
```

### Move the Form Data Type

Originally:

```tsx
type FormData = {
    prompt: string;
};
```

was defined in `ChatBot`.

- Since this type is directly related to the form, move it into `ChatInput`.

### Rename `FormData`

- `FormData` is actually a built-in browser type used with the Fetch API for submitting forms.
- Using the same name could accidentally clash with the built-in type.

Rename it to something more specific:

```tsx
type ChatFormData = {
    prompt: string;
};
```

## Decide What Logic Belongs in `ChatInput`

The existing `onSubmit` function inside `ChatBot` contains several responsibilities:

- Updating messages
- Showing loading indicator
- Showing errors
- Resetting the form
- Making API requests

Question:

Should all of this be moved into `ChatInput`?

**Answer: No.**

### Single Responsibility Principle

- `ChatBot` acts as the **manager** of the conversation.
- It controls the conversation and decides how to respond when the user submits the form.

Its responsibilities include:

- Managing chat messages
- Calling APIs
- Handling responses
- Managing loading state
- Managing errors

These responsibilities should remain inside `ChatBot`.

### What Should Move?

Only the logic directly related to the form belongs inside `ChatInput`.

Specifically:

```tsx
reset({ prompt: "" });
```

Resetting the form is part of form management, so it should stay inside `ChatInput`.

```tsx
onSubmit={handleSubmit(data => {
    reset({ prompt: "" });
})}
```

### Notify the Parent Component

- After resetting the form, `ChatInput` must notify `ChatBot` that a new prompt has been submitted.
- This is done using **props**.

### Create a Props Type

```tsx
type Props = {
    onSubmit: (data: ChatFormData) => void;
};
```

- The component now accepts an `onSubmit` function from its parent.

### Destructure Props

```tsx
function ChatInput({ onSubmit }: Props);
```

- Now `onSubmit` can be called directly.
- `onSubmit` function is passed to this component by the parent or the consumer of this component.

After resetting the form:

```tsx
onSubmit(data);
```

This informs the parent (`ChatBot`) that a prompt has been submitted.

### Extract Submit Logic

```tsx
const handleFormSubmit = handleSubmit((data) => {
    reset({ prompt: "" });
    onSubmit(data);
});
```

### Why `handleFormSubmit`?

React Hook Form already provides:

```tsx
handleSubmit(...)
```

- Using another function named `handleSubmit` would be confusing.

A more descriptive name:

```tsx
handleFormSubmit;
```

### Understanding `handleSubmit`

React Hook Form provides:

```tsx
handleSubmit(callback);
```

It:

1. Validates the form.
2. If valid, calls your callback and passes the form data to the callback.
3. Returns another function.

Example:

```tsx
const submit = handleSubmit(callback);
```

Here:

- `callback` is your logic.
- `submit` is the function that actually gets executed.

## Move Keyboard Logic

Previously, `ChatBot` contained:

```tsx
onKeyDown;
```

Move this function into `ChatInput`.

### Import `KeyboardEvent`

Because `onKeyDown` was moved, import:

```tsx
import { KeyboardEvent } from "react";
```

### Rename the Function

- Rename `onKeyDown` to `handleKeyDown` for consistency with other event handlers.

Use:

```tsx
onKeyDown = { handleKeyDown };
```

### Pass `onSubmit` from Parent

Inside `ChatBot`:

```tsx
<ChatInput onSubmit={onSubmit} />
```

Now `ChatInput` can notify the parent whenever the user submits a prompt.

### Type Error

A type error appears because `onSubmit()` still expects:

```tsx
FormData;
```

but the new component uses:

```tsx
ChatFormData;
```

### Export the Type

Inside `ChatInput`:

```tsx
export type ChatFormData = {
    prompt: string;
};
```

- Import it inside `ChatBot`.

## Component Analogy

A car consists of multiple parts:

- Engine
- Wheels
- Seats

Similarly, `ChatBot` consists of multiple child components:

- Chat Messages
- Typing Indicator
- Chat Input

The parent component acts as the **manager** that brings everything together.

### Should API Logic Be Extracted?

Question:

Should the API call be moved into:

- a service
- a custom hook

**Answer:** It depends.

There is **no universally correct answer**.

### When Not to Extract

- We should **not** to extract the API logic because:

1. The API call is used only in this component.
    - There is no reuse.

2. The logic is very short.
    - Only a few lines of code.
    - Extracting it would add unnecessary files.

3. `ChatBot` itself is only about **50 lines**.
    - Splitting it further provides very little benefit.

### Problem

- When Enter is pressed `handleKeyDown()` executes.
- Inside it `handleSubmit(onSubmit)` is called.
- Notice that this directly calls the parent's `onSubmit`.
- The parent's callback **does not reset the form**.

Therefore `reset(...)` never executes.

### Correct Fix

Instead of:

```tsx
handleSubmit(onSubmit);
```

call:

```tsx
handleFormSubmit();
```

because this function performs **both** actions:

1. Reset the form.
2. Notify the parent.

### Rename the Function

- The name `handleFormSubmit` feels awkward.
- Rename it to `submit`.

Now the code becomes cleaner.

## Final Code

```tsx
import type { KeyboardEvent } from "react";
import { FaArrowUp } from "react-icons/fa";
import { Button } from "../ui/button";
import { useForm } from "react-hook-form";

export type ChatFormData = {
    prompt: string;
};

type Props = {
    onSubmit: (data: ChatFormData) => void;
};

const ChatInput = ({ onSubmit }: Props) => {
    const { register, handleSubmit, reset, formState } =
        useForm<ChatFormData>();

    const submit = handleSubmit((data) => {
        reset({ prompt: "" });
        onSubmit(data);
    });

    const handleKeyDown = (e: KeyboardEvent<HTMLFormElement>) => {
        if (e.key === "Enter" && !e.shiftKey) {
            e.preventDefault();
            submit();
        }
    };

    return (
        <form
            onSubmit={submit}
            onKeyDown={handleKeyDown}
            className="flex flex-col gap-2 items-end border-2 p-4 rounded-2xl"
        >
            <textarea
                {...register("prompt", {
                    required: true,
                    validate: (data) => data.trim().length > 0,
                })}
                autoFocus
                className="w-full border-0 outline-0 resize-none text-sm"
                placeholder="Ask anything"
                maxLength={1000}
            ></textarea>
            <Button
                disabled={!formState.isValid}
                className="cursor-pointer rounded-full w-9 h-9"
            >
                <FaArrowUp />
            </Button>
        </form>
    );
};

export default ChatInput;
```
