# Extracting a ChatMessages Component

- The logic for rendering chat messages should live inside its own component.

Create:

```text
components/chat/ChatMessages.tsx
```

## Initial Implementation

```tsx
import ReactMarkdown from "react-markdown";

const ChatMessages = () => {
    return (
        <div>
            {messages.map((message, index) => (
                <div
                    key={index}
                    onCopy={onCopyMessage}
                    ref={index == messages.length - 1 ? lastMessageRef : null}
                    className={`px-3 py-1 rounded-xl ${
                        message.role === "user" ?
                            "bg-blue-600 text-white self-end"
                        :   "bg-gray-100 text-black self-start"
                    }`}
                >
                    <ReactMarkdown>{message.content}</ReactMarkdown>
                </div>
            ))}
        </div>
    );
};

export default ChatMessages;
```

## Fix 1: Pass the Messages Array

- `messages` belongs to the parent component.
- Since `ChatMessages` doesn't own that state, it must receive it through props.

### Define a Props Type

```tsx
type Props = {
    messages: Message[];
};
```

- However, `Message` currently exists inside `Chatbot.tsx`.
- Move it into `ChatMessages.tsx`.

```tsx
export type Message = {
    content: string;
    role: "user" | "bot";
};
```

Exporting it allows the parent component to reuse the same type.

## Fix 2: Move the Copy Handler

- The `onCopyMessage` function is only used while rendering chat messages.
- Instead of keeping it in `Chatbot.tsx`, move it into `ChatMessages.tsx`.

This keeps all message-related logic together.

## Fix 3: Move the Auto-Scrolling Logic

Previously, the chatbot component contained:

- `lastMessageRef`
- `useEffect`
- Scrolling logic

All of these exist solely because of the messages list.

- Therefore, they belong inside the `ChatMessages` component.
- This is another example of encapsulation.
- Instead of placing unrelated logic inside one large component, every module manages the functionality that directly belongs to it.

## Final Code

```tsx
import { useEffect, useRef } from "react";
import ReactMarkdown from "react-markdown";

export type Message = {
    content: string;
    role: "user" | "bot";
};

type Props = {
    messages: Message[];
};

const ChatMessages = ({ messages }: Props) => {
    const lastMessageRef = useRef<HTMLDivElement | null>(null);

    useEffect(() => {
        lastMessageRef.current?.scrollIntoView({
            behavior: "smooth",
        });
    }, [messages]);

    const onCopyMessage = (e: React.ClipboardEvent): void => {
        const selection = window.getSelection()?.toString().trim();

        if (selection) {
            e.preventDefault();
            e.clipboardData.setData("text/plain", selection);
        }
    };

    return (
        <>
            {messages.map((message, index) => (
                <div
                    key={index}
                    onCopy={onCopyMessage}
                    ref={index === messages.length - 1 ? lastMessageRef : null}
                    className={`px-3 py-1 rounded-xl ${
                        message.role === "user" ?
                            "bg-blue-600 text-white self-end"
                        :   "bg-gray-100 text-black self-start"
                    }`}
                >
                    <ReactMarkdown>{message.content}</ReactMarkdown>
                </div>
            ))}
        </>
    );
};

export default ChatMessages;
```

## Why This Design Is Better

`ChatMessages` now has a single responsibility:

- Rendering chat messages
- Rendering Markdown
- Handling copy behavior
- Automatically scrolling to the latest message

Everything related to displaying messages is encapsulated inside one module.

- This way, each module or component encapsulates all of its related functionality.
- This is one of the key benefits of modularizing our code.

### Use the Component

Finally, render `ChatMessages` inside `Chatbot`.

```tsx
<div className="flex flex-col flex-1 gap-2 my-5 overflow-y-auto">
    <ChatMessages messages={messages} />
    {isBotTyping && <TypingIndicator />}
    {error && <p className="text-red-500">{error}</p>}
</div>
```
