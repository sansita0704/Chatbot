# Defining the Prisma Schema

- The Prisma schema describes the structure of our database directly in code.
- Install the **Prisma** extension in VS Code for syntax highlighting and autocomplete.

We will define **three models**:

1. Product
2. Review
3. Summary

## Model 1: Product

```prisma
model Product {
  id          Int      @id @default(autoincrement())
  name        String
  description String?
  price       Float
}
```

### Fields

Inside a model, we define fields using the syntax:

```text
<field-name> <type> <attributes>
```

Example:

```prisma
id Int @id @default(autoincrement())
```

### Attributes

- Attributes customize the behavior of fields.
- All attributes begin with `@`.

1. `@id`

Marks the field as the **primary key**.

- Uniquely identifies every record in the table.
- No two products can have the same ID.

2. `@default()`

Assigns a default value to a field.

Example:

```prisma
@default(autoincrement())
```

3. `autoincrement()`

- Automatically generates IDs.

Instead of manually assigning IDs, the database generates them:

```
1 2 3 4 ...
```

## Model 2: Review

```prisma
model Review {
  id        Int      @id @default(autoincrement())
  author    String
  rating    Int
  content   String
  createdAt DateTime @default(now())
}
```

### `@default(now())`

- When a new review is inserted into the database, Prisma automatically stores the **current date and time** in this field.

### Creating a Relationship

- A review belongs to a particular product.

To represent this relationship, we need two fields.

```prisma
productId Int
product   Product @relation(fields: [productId], references: [id])
```

### `productId`

- Stores the ID of the product that the review belongs to.
- This will become a **foreign key** in the database.

### `product`

- This field links a review to its corresponding `Product` object.
- The field type is `Product`.

### `@relation`

Used to define relationships between models.

```prisma
@relation(fields: [productId], references: [id])
```

It takes two arguments.

#### 1. `fields`

Specifies which field in the current model stores the foreign key.

#### 2. `references`

Specifies which field in the target model is being referenced.

This means:

```
Review.productId
        ↓
Product.id
```

### Relationship on the Other Side

- Relationships must be defined in **both models**.

Therefore, add this field to the `Product` model.

```prisma
reviews Review[]
```

- This means a product can have **multiple reviews**.
- `Review[]` represents an array of reviews.

## Model 3: Summary

This model stores AI-generated summaries.

```prisma
model Summary {
  id          Int      @id @default(autoincrement())
  productId   Int      @unique
  product     Product  @relation(fields: [productId], references: [id])
  content     String
  generatedAt DateTime @default(now())
  expiresAt   DateTime
}
```

### Product Relationship

Each summary belongs to a single product.

```prisma
productId Int @unique
product Product @relation(fields: [productId], references: [id])
```

### `@unique`

- Ensures that each product can have **only one summary**.
- Without `@unique`, multiple summaries could exist for the same product.

### Updating the Product Model

Since a product can have **at most one summary**, add:

```prisma
summary Summary?
```

- The `?` makes the relationship optional.
- Not every product must have a generated summary.
