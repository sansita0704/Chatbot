# Triggering Summary Generation

Create a state variable to store the newly generated summary.

```tsx
const [summary, setSummary] = useState("");
```

## Create the Handler

The handler should:

1. Make a POST request.
2. Store the generated summary.

```tsx
const handleSummarize = async () => {
    const { data } = await axios.post<SummarizeResponse>(
        `/api/products/${productId}/reviews/summarize`,
    );

    setSummary(data.summary || "");
};
```

### Response Type

```tsx
type SummarizeResponse = {
    summary: string | null;
};
```

### Update the Button

```tsx
<Button onClick={handleSummarize}>
    <HiSparkles />
    Summarize
</Button>
```

## Why Doesn't the UI Update?

- After clicking **Summarize**, the summary is generated successfully, but the UI still displays the button.

While rendering, we're checking `reviewData?.summary`:

```tsx
{
    reviewData?.summary ?
        <p>{reviewData.summary}</p>
    :   <Button onClick={handleSummarize} className="rounded-lg">
            <HiSparkles />
            Summarize
        </Button>;
}
```

- However, `reviewData.summary` contains the summary fetched when the page initially loaded.
- The newly generated summary is stored in a different state variable `summary`.

Therefore, the UI doesn't update automatically.

## Solution

Create a variable that uses whichever summary is available.

```tsx
const currentSummary = reviewData.summary || summary;
```

Update the rendering logic:

```tsx
{
    currentSummary ?
        <p>{currentSummary}</p>
    :   <Button onClick={handleSummarize} className="rounded-lg">
            <HiSparkles />
            Summarize
        </Button>;
}
```

Now, as soon as a summary is generated, the UI updates immediately.
