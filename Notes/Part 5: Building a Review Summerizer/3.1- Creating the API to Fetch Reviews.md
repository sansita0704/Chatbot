# Building the First API Endpoint with Prisma

### Step 1: Create the Route

In `routes.ts`, add a new GET route.

```ts
router.get("/api/products/:id/reviews", (req: Request, res: Response) => {});
```

The `:id` part is a route parameter.

### Step 2: Create a Prisma Client

Inside the route handler:

```ts
const prisma = new PrismaClient();
```

Import it:

```ts
import { PrismaClient } from "@prisma/client";
```

The Prisma Client is automatically regenerated whenever:

- we modify `schema.prisma` and
- create a new migration

This regenerated client always reflects the latest database schema.

### Step 3: Read the Product ID

Extract the route parameter.

```ts
const productId = req.params.id;
```

Type of `req.params.id`:

```ts
string | undefined;
```

Convert it to a number.

```ts
const productId = Number(req.params.id);
```

### Step 4: Fetch Reviews with Prisma

Use:

```ts
const reviews = await prisma.review.findMany({
    where: {
        productId,
    },
});
```

This generates SQL similar to:

```sql
SELECT *
FROM reviews
WHERE productId = @productId
```

With the object that we pass to `findMany()`, we can customize our query.

### Step 5: Sort Reviews

Customize the query using `orderBy`.

```ts
const reviews = await prisma.review.findMany({
    where: {
        productId,
    },
    orderBy: {
        createdAt: "desc",
    },
});
```

Equivalent SQL:

```sql
SELECT *
FROM reviews
WHERE productId = ?
ORDER BY createdAt DESC;
```

Latest reviews appear first.

### Why Use Prisma?

- Instead of writing SQL manually, we write JavaScript/TypeScript objects.
- Prisma automatically generates the SQL queries.

Prisma is **not a replacement for learning SQL**.

- Works great for simple CRUD operations.
- Complex queries often require writing raw SQL manually.

Knowing SQL is still important.

### Step 6: Await the Query

- `findMany()` returns a Promise.

Therefore:

```ts
const reviews = await prisma.review.findMany(...);
```

### Step 7: Prisma Returns Typed Objects

- The returned value is an array of Review objects.

Each object contains fields defined in the Prisma schema, such as:

- id
- author
- rating
- content
- etc.

The types are automatically generated from `schema.prisma`.

### Step 8: Send the Response

Return the reviews as JSON.

```ts
res.json(reviews);
```

## Problem: Non-Numeric Product ID

Example request:

```
GET /api/products/abc/reviews
```

Without validation:

- Prisma throws an error.
- Server returns:

```
500 Internal Server Error
```

A 500 error means the backend crashed, which should be avoided.

### Step 9: Validate the Product ID

Check whether the converted number is valid:

```ts
if (isNaN(productId)) {
    ...
}
```

### Two Possible Approaches

1. Return an empty array.

```json
[]
```

2. Return a **400 Bad Request**.

```json
{
    "error": "Invalid product ID"
}
```

### Why Return 400 Instead of an Empty Array?

- A non-numeric product ID generally indicates a problem.

Two likely scenarios:

### Scenario 1: A Hacker

Someone manually sends requests like:

```
/api/products/abc/reviews
```

Returning either an empty array or an error makes little difference.

### Scenario 2: Bug in the Frontend (More Common)

- Suppose the frontend incorrectly extracts the product ID.
- Instead of `1`, it accidentally sends an object.

Resulting URL:

```
/api/products/[object Object]/reviews
```

If the backend returns:

```json
[]
```

developers might incorrectly assume:

- the request was valid
- the product simply has no reviews

The real bug remains hidden.

- Returning an actual error immediately reveals that the frontend is sending an invalid request.
- This makes debugging much easier.

### Returning a 400 Error

Implementation:

```ts
if (isNaN(productId)) {
    res.status(400).json({
        error: "Invalid product ID",
    });

    return;
}
```

The `return` is important because it prevents the remaining code from executing.

## Final Route

```ts
router.get("/api/products/:id/reviews", async (req: Request, res: Response) => {
    const prisma = new PrismaClient();
    const productId = Number(req.params.id);

    if (isNaN(productId)) {
        res.status(400).json({
            error: "Invalid product ID",
        });
        return;
    }

    const reviews = await prisma.review.findMany({
        where: { productId },
        orderBy: { createdAt: "desc" },
    });

    res.json(reviews);
});
```
