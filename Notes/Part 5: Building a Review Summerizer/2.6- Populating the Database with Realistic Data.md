# Populating the Database with Realistic Data

- We can manually insert sample data into the database.
- However, a faster and smarter approach is to use ChatGPT to generate realistic sample data.

Use the following prompt:

```
Here's my prisma schema:
<code of schema.prisma>

Generate a complete SQL script to populate the products and reviews table in a MYSQL database based on the schema above.

- Create 5 products
- For each product, insert 5 realistic customer reviews.
- Make sure each review is long and tailored to the product type.
- Do not include data for the summaries table.
- Output only the SQL script. No comments or explanations.
```

### Why include the Prisma schema?

By including:

```
Here's my prisma schema:
```

followed by the contents of `schema.prisma`, we give ChatGPT the necessary context to generate SQL that matches our database structure.

### Prompt Breakdown

Main instruction:

```
Generate a complete SQL script to populate the products and reviews table in a MySQL database based on the schema above.
```

The order of the instructions in the prompt does not matter.

### Important Points

1. We do **not** populate the `Summary` table because summaries will be generated at runtime using the AI model.

2. Specify that the target database is **MySQL** so the generated SQL uses the correct syntax.

3. Emphasize that the SQL should be **based on the schema above** to ensure it matches our database structure.

Once ChatGPT generates the SQL script, simply run it against the database to insert the sample data.
