# Extracting the API Layer

- Currently, our component is responsible for making HTTP requests using Axios.
- A better approach is to move all backend communication into a dedicated **API layer**.

Benefits:

- Components no longer need to import or use Axios directly.
- All HTTP request logic lives in one place.
- Components focus only on rendering UI and managing state.

## Create a New File

```text
reviews/reviewsApi.ts
```

```ts
import axios from "axios";

type Review = {
    id: number;
    author: string;
    content: string;
    rating: number;
    createdAt: string;
};

export type GetReviewsResponse = {
    summary: string | null;
    reviews: Review[];
};

export type SummarizeResponse = {
    summary: string | null;
};

export const reviewsApi = {
    async fetchReviews(productId: number) {
        const { data } = await axios.get<GetReviewsResponse>(
            `/api/products/${productId}/reviews`,
        );

        return data;
    },

    async summarizeReviews(productId: number) {
        const { data } = await axios.post<SummarizeResponse>(
            `/api/products/${productId}/reviews/summarize`,
        );

        return data;
    },
};
```

### Why Place It Inside the `reviews` Directory?

- This module only contains logic related to the **Reviews** feature.
- It is not shared with other parts of the application.
- Keeping it inside the `reviews` directory groups all related code together, making the project easier to navigate and maintain.

### Responsibilities of `reviewsApi.ts`

This module contains:

- All review-related API calls.
- The response types returned by the backend.
- The logic for communicating with the backend.

## Update `ReviewList.tsx`

Import the API layer.

```tsx
import {
    reviewsApi,
    type GetReviewsResponse,
    type SummarizeResponse,
} from "./reviewsApi";
```

### Update the Mutation

```tsx
const summaryMutation = useMutation<SummarizeResponse>({
    mutationFn: () => reviewsApi.summarizeReviews(productId),
});
```

It is now clear that the mutation uses the API layer to generate the review summary.

### Update the Query

```tsx
const reviewsQuery = useQuery<GetReviewsResponse>({
    queryKey: ["reviews", productId],
    queryFn: () => reviewsApi.fetchReviews(productId),
});
```

The query also delegates the backend request to the API layer.

## Final Result

After this refactoring, `ReviewList.tsx` is primarily responsible for:

1. Creating the query.
2. Creating the mutation.
3. Rendering the UI.

All backend communication has been moved into the dedicated API layer, resulting in cleaner, more maintainable code.
