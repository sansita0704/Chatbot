# Handling Regeneration

- Even after storing the summary, we are still regenerating it every time the API is called.

Instead, we should:

- Generate a summary only if one **does not already exist**, or
- Generate a new summary if the existing one has **expired**.

To do this, we first need to check whether a summary already exists for the product.

## Create a New Repository Function

```ts
getReviewSummary(productId: number) {
    return prisma.summary.findUnique({
        where: { productId },
    });
},
```

We use `findUnique()` because we are retrieving a single summary.

## Check for an Existing Summary

In the service:

```ts
const existingSummary = await reviewRepository.getReviewSummary(productId);

if (existingSummary && existingSummary.expiresAt > new Date())
    return existingSummary.content;
```

### Flow

1. Retrieve the existing summary.
2. Check whether it exists.
3. Compare its `expiresAt` value with the current date.
4. If the summary has **not expired**, return the cached summary instead of generating a new one.

This avoids unnecessary AI calls and saves both time and token usage.
