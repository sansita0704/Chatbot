# Refactoring for Readability

Instead of destructuring everything returned by `useMutation()`, we can keep the returned object intact.

```tsx
const summaryMutation = useMutation<SummarizeResponse>({
    mutationFn: () => summarizeReviews(),
});
```

This improves readability because every mutation-related property is accessed through a single object.

## Update the Code

Current summary:

```tsx
const currentSummary = reviewData.summary || summaryMutation.data?.summary;
```

Button:

```tsx
<Button
    onClick={() => summaryMutation.mutate()}
    className="rounded-lg cursor-pointer"
    disabled={summaryMutation.isPending}
>
    <HiSparkles />
    {summaryMutation.isPending ? "Summarizing..." : "Summarize"}
</Button>
```

Error message:

```tsx
{
    summaryMutation.isError && (
        <p className="mt-2 text-red-500">
            Could not summarize the reviews. Try again!
        </p>
    );
}
```

- Using a single object (`summaryMutation`) makes it immediately clear where these properties come from.
- The same refactoring can also be applied to the **query hook**.
