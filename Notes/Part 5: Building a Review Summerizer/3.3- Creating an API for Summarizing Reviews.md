# Creating an API for Summarizing Reviews

Create the following route:

```ts
router.post(
    "/api/products/:id/reviews/summarize",
    reviewController.summarizeReviews,
);
```

- Since this endpoint creates new data (a generated summary), we use the **POST** HTTP method.
- We append `/summarize` to the route to follow RESTful conventions.

Next, create a new controller method to handle the request.

## Controller

```ts
async summarizeReviews(req: Request, res: Response) {
    const productId = Number(req.params.id);

    if (isNaN(productId)) {
        res.status(400).json({
            error: 'Invalid product ID',
        });
        return;
    }

    const summary = await reviewService.summarizeReviews(productId);

    res.json({ summary });
},
```

## Getting the Last 10 Reviews

- Generating a summary is the responsibility of the **service layer**.

- The service first retrieves the **last 10 reviews** and sends them to the AI model.

```ts
const reviews = await reviewRepository.getReviews(productId, 10);
```

### Why Only the Last 10 Reviews?

- Although we could send any number of reviews to the AI model, we intentionally avoid sending all of them.

There are two reasons for this:

#### 1. Reduce the Context Window

- As the number of reviews grows, a product could eventually have hundreds or even thousands of reviews.
- Sending all of them to the AI model would unnecessarily increase the prompt size and consume more of the model's context window.

#### 2. More Relevant Information

- From a user's perspective, the **most recent reviews** are usually more useful because they better reflect the product's current quality.

## Updating the Review Repository

Update the repository method to accept an optional `limit` parameter.

```ts
async getReviews(productId: number, limit?: number): Promise<Review[]> {
    const prisma = new PrismaClient();

    return prisma.review.findMany({
        where: { productId },
        orderBy: { createdAt: "desc" },
        take: limit,
    });
}
```

- The optional `limit` parameter is passed directly to Prisma's `findMany()` method using the `take` option.

## Joining the Reviews into a Single String

Before sending the reviews to the AI model, we combine them into one large string.

### Step 1: Extract the review content

```ts
reviews.map((r) => r.content);
```

### Step 2: Join all reviews

```ts
const joinedReviews = reviews.map((r) => r.content).join("\n\n");
```
