# Displaying Loading Skeletons

Create a state variable to track whether the summary is currently being generated.

```tsx
const [isSummaryLoading, setIsSummaryLoading] = useState(false);
```

### Update `handleSummarize()`

```tsx
const handleSummarize = async () => {
    setIsSummaryLoading(true);

    const { data } = await axios.post<SummarizeResponse>(
        `/api/products/${productId}/reviews/summarize`,
    );

    setSummary(data.summary || "");
    setIsSummaryLoading(false);
};
```

### Flow

1. Set `isSummaryLoading` to `true`.
2. Make the API request.
3. Save the generated summary.
4. Set `isSummaryLoading` back to `false`.

### Reusing the Review Skeleton

- The loading skeleton used while fetching reviews can also be reused while generating summaries.

Create a new file:

```text
reviews/ReviewSkeleton.tsx
```

```tsx
import Skeleton from "react-loading-skeleton";

const ReviewSkeleton = () => {
    return (
        <div>
            <Skeleton width={150} />
            <Skeleton width={100} />
            <Skeleton count={2} />
        </div>
    );
};

export default ReviewSkeleton;
```

## Update `ReviewList.tsx`

### While Fetching Reviews

Replace the inline skeletons with the reusable component.

```tsx
if (isLoading)
    return (
        <div className="flex flex-col gap-5">
            {[1, 2, 3].map((i) => (
                <ReviewSkeleton key={i} />
            ))}
        </div>
    );
```

### While Generating the Summary

```tsx
{
    currentSummary ?
        <p>{currentSummary}</p>
    :   <div>
            <Button
                onClick={handleSummarize}
                className="rounded-lg cursor-pointer"
                disabled={isSummaryLoading}
            >
                <HiSparkles />
                {isSummaryLoading ? "Summarizing..." : "Summarize"}
            </Button>

            {isSummaryLoading && (
                <div className="py-3">
                    <ReviewSkeleton />
                </div>
            )}
        </div>;
}
```

### Important

- While the summary is being generated, the **Summarize** button should be disabled.
- Otherwise, users could click it multiple times and send multiple API requests.

That's why we add:

```tsx
disabled = { isSummaryLoading };
```
