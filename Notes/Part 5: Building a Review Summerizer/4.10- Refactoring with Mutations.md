# Refactoring with Mutations

- Earlier, we used the **query hook** to fetch data.

However:

- `useQuery` is intended for **reading (querying)** data.
- If we want to **create** or **update** data, we should use **`useMutation`**.

### 1. Clean Up `handleSummarize()`

Since the mutation hook will handle loading and error states for us:

- Remove all state update logic.
- Remove the `try...catch` block as error handling is implemented inside mutation hook.
- Rename `handleSummarize()` to `summarizeReviews()` because it no longer handles the button click directly. Its only responsibility is to call the backend.

```tsx
const summarizeReviews = async () => {
    const { data } = await axios.post<SummarizeResponse>(
        `/api/products/${productId}/reviews/summarize`,
    );

    return data;
};
```

The function now simply:

1. Calls the backend.
2. Returns the response.

### 2. Remove State Variables

The mutation hook provides built-in loading, error, and response states.

Therefore, remove:

```tsx
const [summary, setSummary] = useState("");
const [isSummaryLoading, setIsSummaryLoading] = useState(false);
const [summaryError, setSummaryError] = useState("");
```

### 3. Use the `useMutation` Hook

`useMutation()` is a generic function, so we specify the type returned by the backend.

```tsx
const {
    mutate: handleSummarize,
    isPending: isSummaryLoading,
    isError: isSummaryError,
    data: summarizeResponse,
} = useMutation<SummarizeResponse>({
    mutationFn: () => summarizeReviews(),
});
```

We pass an object with the following property:

### `mutationFn`

```tsx
mutationFn: () => summarizeReviews();
```

This specifies the function responsible for performing the mutation.

### What Does `useMutation()` Return?

`useMutation()` returns an object containing several useful properties.

```tsx
const {
    mutate: handleSummarize,
    isPending: isSummaryLoading,
    isError: isSummaryError,
    data: summarizeResponse,
} = useMutation<SummarizeResponse>(...);
```

### `mutate`

This function is called when the **Summarize** button is clicked.

### `isPending`

Becomes `true` while the mutation request is in progress.

### `isError`

Becomes `true` if the mutation fails.

### `data`

Contains the response returned by the backend.

> **Note:** We renamed these properties so they match the naming used throughout our component and avoid naming conflicts.

### 4. Update the Remaining Code

Update the current summary:

```tsx
const currentSummary = reviewData.summary || summarizeResponse?.summary;
```

Update the button:

```tsx
<Button
    onClick={() => handleSummarize()}
    className="rounded-lg cursor-pointer"
    disabled={isSummaryLoading}
>
```

Render the error:

```tsx
{
    isSummaryError && (
        <p className="mt-2 text-red-500">
            Could not summarize the reviews. Try again!
        </p>
    );
}
```
