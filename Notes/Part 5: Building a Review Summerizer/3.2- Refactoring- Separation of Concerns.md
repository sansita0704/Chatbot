# Refactoring the Reviews API

## Step 1: Introduce a Controller

Create a new file:

```text
controllers/review.controller.ts
```

Export an object:

```ts
export const reviewController = {
    async getReviews(req: Request, res: Response) {},
};
```

Import the Express types:

```ts
import type { Request, Response } from "express";
```

### Move Route Logic into the Controller

Move the entire route handler code from `routes.ts` into:

```ts
reviewController.getReviews();
```

Since it uses `await`, mark the method as `async`.

### Update Routes

```ts
router.get("/api/products/:id/reviews", reviewController.getReviews);
```

### Problem with the Controller

Although the route is cleaner, the controller still contains database access code.

For example:

- creating Prisma client
- querying the database
- sorting reviews

This logic does **not** belong in a controller.

- Controllers should only deal with HTTP concerns.

## Step 2: Introduce a Service

Create:

```text
services/review.service.ts
```

```ts
export const reviewService = {
    async getReviews(productId: number): Promise<Review[]> {},
};
```

Import the generated Prisma `Review` type.

### Move Database Logic to the Service

Move the following from the controller:

- Prisma client
- database query
- sorting logic

into the service.

Mark the method as `async`.

### Return Reviews

Initially the code looked like:

```ts
const reviews = await prisma.review.findMany(...);

return reviews;
```

This can be simplified.

Instead write:

```ts
return prisma.review.findMany(...);
```

Reason:

- `findMany()` already returns a Promise.
- Since the service immediately returns the result, there is no need to await it first.

This makes the method shorter and cleaner.

### Update the Controller

Controller simply calls the service.

```ts
const reviews = await reviewService.getReviews(productId);
res.json(reviews);
```

### Problem with the Service

- The service still contains database access code.
- Database access should not live inside a service.

It belongs to a **Repository**.

## Step 3: Introduce a Repository

Create:

```text
repositories/review.repository.ts
```

Export an object.

```ts
export const reviewRepository = {
    async getReviews(productId: number): Promise<Review[]> {
        const prisma = new PrismaClient();

        return prisma.review.findMany({
            where: { productId },
            orderBy: { createdAt: "desc" },
        });
    },
};
```

The repository's responsibility is **only data access**.

### Update the Service

- Remove the Prisma code.
- Instead call the repository.

```ts
return reviewRepository.getReviews(productId);
```

## Current Service Behavior

The service currently looks like:

```ts
return reviewRepository.getReviews(productId);
```

It is simply acting as a pass-through.
