# Setting Up Prisma

To connect our application to the database, we will use **Prisma**.

## Prisma

- Prisma is an **ORM (Object Relational Mapper)**.
- It acts as a layer between our application and the database.
- Think of it as a **translator** between our code and SQL.
- It allows our application to communicate with the database in a **clean**, **type-safe**, and **developer-friendly** way.

## Installing Prisma

Inside the **server** directory, install the required packages.

```bash
bun add -d prisma
bun add @prisma/client
```

### `prisma`

- Installed as a **development dependency**.
- Provides the **Prisma CLI**.
- Used to:
    - Initialize Prisma
    - Define the database schema
    - Create and run migrations
    - Generate the Prisma Client

### `@prisma/client`

- Installed as a **runtime dependency**.
- It is the generated client that our application uses to interact with the database.

### Initializing Prisma

Inside the **server** directory, run:

```bash
bunx prisma init
```

This creates a new **prisma** folder containing a file named `schema.prisma`.

```prisma
generator client {
  provider = "prisma-client"
  output   = "../generated/prisma"
}

datasource db {
  provider = "mysql"
}
```

The file has two sections:

1. **Defining Client**
2. **Defining Database**

### Configuration Changes

1. Set the database provider

Inside the `datasource` block:

```prisma
provider = "mysql"
```

2. Configure the database connection

Open the `.env` file and add:

```env
DATABASE_URL="mysql://root:password@localhost:3306/review_summarizer"
```

### Breakdown

```
mysql://root:password@localhost:3306/review_summarizer
```

| Part                | Meaning                         |
| ------------------- | ------------------------------- |
| `mysql`             | Database type                   |
| `root`              | MySQL username                  |
| `password`          | Password used to log into MySQL |
| `localhost`         | Database server                 |
| `3306`              | Default MySQL port              |
| `review_summarizer` | Database name                   |

This tells Prisma to connect to the local MySQL server using the specified credentials and database.
