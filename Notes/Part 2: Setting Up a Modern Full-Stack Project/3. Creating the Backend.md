# Creating the Server Application

Navigate to the `server` directory and initialize a new Bun project:

```bash
bun init
```

## Install Express

```bash
bun add express
```

### Node.js Approach

Typically, Node.js uses separate tools:

- `node` → runs JavaScript code
- `npm` → installs dependencies

### Bun Approach

Bun combines these responsibilities into a single tool:

- `bun` → runs code, installs packages, manages scripts, and more

### Install Express Type Definitions

Since we are using TypeScript, install Express type definitions:

```bash
bun add -d @types/express # as a dev dependency
```

#### Why?

TypeScript needs type information for Express objects such as:

- `Request`
- `Response`
- `Application`

This package provides those type definitions.

### Multiple package.json Files

At this point, we have two `package.json` files:

1. In root directory
2. In server directory

Each workspace manages its own dependencies while still being part of the larger project.

## Create a Basic Web Server

Inside `server/index.ts`:

```ts
import express from "express";
import type { Request, Response } from "express";

const app = express();
const port = process.env.PORT || 3000;

// Define a route
app.get("/", (req: Request, res: Response) => {
    res.send("Hello World!");
});

// Start the server
app.listen(port, () => {
    console.log(`Server is running on http://localhost:${port}`);
});
```

## Understanding the Code

### 1. Import Statements

```ts
import express from "express";
```

- Imports the default export from the Express package.

```ts
import type { Request, Response } from "express";
```

- Imports only the TypeScript types for request and response objects.
- Using `import type` ensures these imports are removed during compilation because they are used only for type checking.

### 2. Creating the Express Application

```ts
const app = express();
```

- The `express()` function creates an object.
- The returned `app` object is used to:
    - Define routes
    - Configure middleware
    - Start the server

### 3. Defining the Port

```ts
const port = process.env.PORT || 3000;
```

This statement uses:

- `process.env.PORT` if it exists
- Otherwise, falls back to `3000`

### 4. Defining a Route

```ts
app.get("/", (req, res) => {
    res.send("Hello World!");
});
```

`app.get()` registers a route handler for HTTP GET requests.

It accepts:

1. Route path

```
"/"
```

2. Callback function

```ts
(req, res) => {};
```

- The callback executes whenever a request is received on that route.

#### Parameters

1. Request (`req`)

Contains information about the incoming request:

- Headers
- Query parameters
- Route parameters
- Request body

2. Response (`res`)

Used to send data back to the client.

Example:

```ts
res.send("Hello World!");
```

sends a text response to the browser.

### 5. Starting the Server

```ts
app.listen(port, () => {
    console.log(`Server is running on http://localhost:${port}`);
});
```

`app.listen()` starts the HTTP server.

It accepts:

1. Port number
2. Callback function

The callback executes once the server has successfully started.

## Running the Server

Run the server directly:

```bash
bun run index.ts
```

### Adding Scripts

Inside `package.json`:

```json
{
    "scripts": {
        "start": "bun run index.ts",
        "dev": "bun --watch run index.ts"
    }
}
```

The `--watch` flag tells Bun to:

- Monitor project files for changes
- Automatically restart the application when changes are detected

This provides functionality similar to **nodemon**.

### Why `bun run dev` Instead of `bun dev`?

- `dev` is a custom script defined inside `package.json`.

Therefore:

```bash
bun run dev
```

works because Bun looks inside the `scripts` section.

```bash
bun dev
```

does not work automatically because `dev` is not a built-in Bun command.

#### Built-in Command

```bash
bun start
```

is recognized by Bun as a special script shortcut.

#### Custom Command

```bash
bun run dev
```
