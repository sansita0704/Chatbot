# Prettier

- **Prettier** is a code formatter.
- It automatically formats code according to predefined style rules.

## Defining Formatting Rules

Create a file named `.prettierrc`:

```json
{
    "singleQuote": true,
    "semi": true,
    "trailingComma": "es5",
    "printWidth": 80,
    "tabWidth": 4
}
```

## Understanding the Configuration

### `singleQuote`

- Converts double quotes to single quotes.

> Note: JSON itself does not support single quotes. This option affects only JavaScript, TypeScript, JSX, etc.

### `semi`

- Adds semicolons.

### `trailingComma`

- Determines whether trailing commas should be added.

Example:

```js
const user = {
    name: "John",
    age: 20,
};
```

- Notice the comma after the last property.

With `"es5"`:

- Adds trailing commas where ES5 allows them.
- Common locations:
    - Objects
    - Arrays
    - Function calls

### `printWidth`

- Preferred maximum line length before Prettier wraps code.

### `tabWidth`

- Indentation size.

## Formatting Code from the Command Line

Formatting from the command line is useful before:

- Pushing code to GitHub
- Creating pull requests
- Sharing code with teammates

### Install Prettier

Install it as a development dependency from the root directory:

```bash
bun add -d prettier
```

**Why install it in the root?**

Because we want Prettier to be available for the entire monorepo, including:

- Client
- Server
- Root files

### Add a Formatting Script

Inside the root `package.json`:

```json
{
    "scripts": {
        "dev": "bun run index.ts",
        "format": "prettier --write ."
    }
}
```

- `--write` → modifies files directly.
- `.` → start formatting from the current directory and recursively process files.

### Ignoring Files During Formatting

We do not want Prettier to format:

- Third-party packages
- Generated files
- Lock files

Create `.prettierignore`:

```text
node_modules
bun.lock
```

This works similarly to `.gitignore`.

### Run Formatting

```bash
bun run format
```

Prettier will:

1. Scan files.
2. Skip ignored files.
3. Format all supported files.
