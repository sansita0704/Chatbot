# Connecting Frontend and Backend

To communicate between the frontend and backend, we first create an API endpoint in the server application.

## Server Side

Inside `server/index.ts`:

```ts
app.get("/api/hello", (req: Request, res: Response) => {
    res.json({
        message: "Hello World!",
    });
});
```

### What's Different Here?

- Previously, we used `res.send(...)` which sends plain text.
- Now we use `res.json(...)` which sends JSON data.

Response:

```json
{
    "message": "Hello World!"
}
```

## Client Side

Inside `App.tsx`:

```tsx
import { useEffect, useState } from "react";

function App() {
    const [message, setMessage] = useState("");

    // Make an API call to "/api/hello"
    useEffect(() => {
        fetch("/api/hello")
            .then((res) => res.json())
            .then((data) => setMessage(data.message));
    }, []);

    return <p>{message}</p>;
}

export default App;
```

### Why Does This Fail?

The request:

```tsx
fetch("/api/hello");
```

is sent from the frontend application.

- The frontend runs on a different server (Vite), which does not contain the `/api/hello` endpoint.
- The endpoint exists only in the backend application.

Therefore, the request fails.

### Solving the Problem with a Proxy

- We configure Vite to forward API requests to the backend server.

Update `vite.config.ts`:

```ts
import { defineConfig } from "vite";
import react from "@vitejs/plugin-react";

export default defineConfig({
    plugins: [react()],

    // Add this server property:
    server: {
        proxy: {
            "/api": "http://localhost:3000",
        },
    },
});
```

### How the Proxy Works

- Whenever Vite receives a request starting with `/api` it automatically forwards the request to:

```text
http://localhost:3000
```

### Example

Frontend request:

```text
/api/hello
```

gets forwarded to:

```text
http://localhost:3000/api/hello
```
