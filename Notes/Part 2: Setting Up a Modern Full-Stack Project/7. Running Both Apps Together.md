# Running Both Applications Together

With the current setup, we need two separate terminal windows:

### Terminal 1

```bash
cd packages/server
bun run dev
```

### Terminal 2

```bash
cd packages/client
bun run dev
```

Managing multiple terminals becomes inconvenient.

## Running Everything with a Single Command

We can use the **concurrently** package.

### Install concurrently

From the root directory:

```bash
bun add -d concurrently
```

### Create Root Entry File

Create `index.ts` in the root directory:

```ts
import concurrently from "concurrently";

concurrently([
    {
        name: "server",
        command: "bun run dev", // to start the server app
        cwd: "packages/server",
        prefixColor: "cyan", // optional
    },
    {
        name: "client",
        command: "bun run dev",
        cwd: "packages/client",
        prefixColor: "green",
    },
]);
```

### Understanding the Configuration

#### `concurrently()`

```ts
concurrently([...]);
```

- Accepts an array of commands that should run simultaneously.
- Each object represents one process.

#### `name`

Label displayed in the terminal output.

#### `command`

Command that should be executed.

#### `cwd`

- **cwd = Current Working Directory**
- The command will run from this directory.
- Without it, Bun would try to execute the command from the root directory.

#### `prefixColor`

- Assigns a color to log messages.
- This makes it easy to distinguish outputs when both applications share the same terminal.

### Add a Root Script

Inside the root `package.json`:

```json
{
    "scripts": {
        "dev": "bun run index.ts"
    }
}
```

### Start the Entire Application

From the root directory:

```bash
bun run dev
```
