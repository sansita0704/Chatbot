# Counting Tokens Programmatically

Developers can estimate token usage before sending requests.

### OpenAI Tokenizer Library

- A popular library is **tiktoken**.
- It uses the same tokenization system as OpenAI models.

## Setting Up a Token Counting Project

### Step 1: Create a Project

```bash
mkdir playground
cd playground
npm init -y
```

This creates a `package.json` file.

### Step 2: Install Tiktoken

```bash
npm install tiktoken
```

### Step 3: Create a File

Create `index.js` file.

## Basic Token Counting Code

```javascript
import { get_encoding } from "tiktoken";

const encoding = get_encoding("cl100k_base");

const tokens = encoding.encode("Hello world, this is a test of tiktoken.");

console.log(tokens);
```

### Understanding Encodings

```javascript
cl100k_base;
```

Meaning:

- `cl` = Chat Language
- `100k` = approximately 100,000 token entries

An encoding acts like a dictionary:

```text
Token ID → Actual Token
```

Example:

```text
904 → "hello"
```

### ES Modules vs CommonJS

Running the code may produce:

```text
Cannot use import statement outside a module
```

### Reason

Node.js defaults to CommonJS.

CommonJS imports use:

```javascript
const { get_encoding } = require("tiktoken");
```

Modern JavaScript uses ES Modules:

```javascript
import { get_encoding } from "tiktoken";
```

### Solution

In `package.json`:

```json
{
    "type": "module"
}
```

This tells Node.js to use ES Modules.

## Output of Tokenization

Running:

```bash
node index.js
```

returns an array like:

```javascript
[
  9906,
  1917,
  0,
  ...
]
```

These numbers are:

- Token IDs
- Not the actual text

Each token ID maps to a specific token in the tokenizer dictionary.
