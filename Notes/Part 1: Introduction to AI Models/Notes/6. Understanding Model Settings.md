# Introduction to Model Settings

Language models provide several settings that control:

- Response format
- Creativity and randomness
- Response length
- Probability selection
- Logging and debugging

Understanding these settings helps developers generate outputs that are better suited to their applications.

## Response Formats

One of the most important settings is **Response Format**.

Common options include:

1. Text
2. JSON Object
3. JSON Schema

### Plain Text Format

Use Cases:

- Chatbots
- Summaries
- General conversation
- Content generation

### JSON Object Format

Prompt:

```text
Give me three benefits of exercising as a JSON object.
```

Example Output:

```json
{
    "benefits_of_exercising": [
        "Improves physical health",
        "Boosts energy",
        "Reduces stress"
    ]
}
```

### JSON Schema Format

- Sometimes developers need responses in a very specific structure.
- This is where **JSON Schema** is useful.

#### Example Desired Structure

```json
{
    "exercise": {
        "benefits": ["Benefit 1", "Benefit 2", "Benefit 3"]
    }
}
```

Instead of hoping the model follows instructions, we provide a schema describing the exact format.

## Temperature

Temperature controls:

> How creative or random the model's responses are.

### Range

```text
0 → 2
```

### Low Temperature

Typical Range:

```text
0.2 – 0.4
```

#### Characteristics

- More predictable
- More focused
- More factual
- Less creative

#### Best For

- Summarization
- Data extraction
- Classification
- Question answering

### High Temperature

Typical Range:

```text
0.7 – 1.0
```

#### Characteristics

- More creative
- More varied
- More expressive
- Less predictable

#### Best For

- Story writing
- Brainstorming
- Marketing content
- Creative writing

### Extreme Temperature Values

#### Temperature = 2

May produce:

- Nonsensical text
- Incoherent sentences
- Unstable outputs
- Gibberish

### Recommended Temperature

A good default value is:

```text
Temperature = 0.7
```

Why?

- Balanced creativity
- Balanced accuracy
- Suitable for many applications

## Max Tokens

Max Tokens controls:

> The maximum length of the generated response.

### Why Max Tokens Matters

Setting an appropriate limit:

#### Prevents

- Excessive costs
- Unnecessarily long responses

### Helps Control

- Output length
- API usage
- Response consistency

### Example: Low Max Tokens

Setting:

```text
Max Tokens = 50
```

Prompt:

```text
Write a story about a robot.
```

Possible Result:

```text
Once upon a time there was a robot named...
```

The response may stop abruptly because the token limit is reached.

#### Prompt Engineering Solution

Prompt:

```text
Write a story about a robot in 50 tokens or less.
```

This helps the model:

- Plan its response
- Finish naturally
- Avoid mid-sentence cutoffs

## Top P

- Top P is another setting that controls randomness.
- It works differently from temperature.

### How Top P Works

Imagine the model predicts several possible next words.

Example:

| Word | Probability |
| ---- | ----------- |
| the  | 40%         |
| a    | 25%         |
| an   | 15%         |
| this | 10%         |
| one  | 10%         |

### Top P = 1

- Uses the full probability distribution.
- The model can choose from all possibilities.

Result:

- More diverse outputs

### Top P = 0.3

Only considers the most probable words.

Result:

- More focused outputs
- Less randomness

### Best Practice

Generally:

- Use Temperature OR Top P
- Avoid heavily adjusting both simultaneously

Recommended:

```text
Temperature = 0.7
Top P = 1
```
