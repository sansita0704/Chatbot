## Creating an API Key

### Steps

1. Open OpenAI Settings.
2. Navigate to **API Keys** (https://platform.openai.com/api-keys).
    >
3. Click **Create New Secret Key**.
4. Give the key a name.
5. Assign it to a project.
6. Copy and save the generated key.

### Security Best Practice

**Never Store API Keys in Source Code**

Example (not recommended):

```javascript
const OPENAI_API_KEY = 'your-api-key';
```

### Why?

Anyone with access to the source code can:

- Use the API key
- Generate requests
- Consume credits
- Increase our costs

### Recommended Approach

- Store API keys in Environment Variables.

Example:

```env
OPENAI_API_KEY=your-api-key
```

Benefits:

- Better security
- Easier deployment
- Prevents accidental exposure

Install the library:

```bash
npm install openai
```

### What the SDK Does

- The library acts as a wrapper around the OpenAI API and simplifies making requests.

The OpenAI SDK:

- Simplifies API calls
- Handles HTTP requests
- Provides convenient methods
- Reduces boilerplate code

Behind the scenes:

```text
Application
     ↓
OpenAI SDK
     ↓
HTTP Request
     ↓
OpenAI API
```


### Creating a Client

The client acts as the connection to OpenAI services.

```javascript
const client = new OpenAI({
    apiKey: OPENAI_API_KEY,
});
```

### Sending a Request

Use:

```javascript
client.responses.create();
```

to generate model responses.

Example:

```javascript
const response = await client.responses.create({
    model: 'gpt-4.1',
    input: 'Write a story about a robot',
    temperature: 0.7,
    max_output_tokens: 50,
});
```

## Useful Response Properties

The response object contains more than just text.

### Output Text

Generated content.

```javascript
response.output_text;
```

### Usage

Token statistics.

```javascript
response.usage;
```

Includes:

- Input tokens
- Output tokens
- Total tokens

### Temperature

Temperature used for generation.

### Top P

Probability setting used.

### Store

Indicates whether logging is enabled.

### Response ID

Unique identifier for the generated response.

Useful for:

- Tracking requests
- Managing conversations
- Debugging

#### Why Response IDs Matter

Each response has a unique ID.

Example uses:

- Conversation memory
- Chatbot state management
- Request tracking

## Introduction to Streaming

By default:

```text
Request → Wait → Entire Response Arrives
```

This means users wait until the full response is generated.

### Enabling Streaming

Set:

```js
stream: true;
```

Example:

```js
const stream = await client.responses.create({
    model: 'gpt-4.1',
    input: 'Write a story about a robot',
    stream: true,
});
```

### What Changes With Streaming?

Without Streaming:

```js
response;
```

With Streaming:

```js
stream;
```

Instead of receiving one complete response, we receive multiple events.

### Async Iterables

- A stream is an async iterable.

Meaning:

- Data arrives over time
- Events can be iterated one by one
- Uses asynchronous iteration

### Iterating Through a Stream

Example:

```js
for await (const event of stream) {
    console.log(event);
}
```

#### Why `for await`?

- Because events arrive gradually rather than all at once.

## Event Objects

- Each stream event contains information about the generation process.

Example properties:

```javascript
{
  type: "response.output_text.delta",
  sequence_number: 252,
  delta: "robot"
}
```

### Event Types

One important type is:

```text
response.output_text.delta
```

This represents:

- A chunk of generated text
- Often a token or partial token

### Delta Property

The actual generated text appears in:

```javascript
event.delta;
```

Example:

```javascript
{
    delta: 'robot';
}
```

### Why Undefined Appears

Not every event contains:

```javascript
delta;
```

Some events represent:

- Start of generation
- End of generation
- Metadata

These events may not contain text.

### Filtering Events

Before printing:

```javascript
if (event.delta) {
    console.log(event.delta);
}
```

This prevents:

```text
undefined
undefined
undefined
```

from appearing.

### Problem With console.log()

`console.log()` automatically adds:

```text
New Line (\n)
```

after each token.

Result:

```text
Hello
world
this
is
a
story
```

### Using process.stdout.write()

To print tokens continuously:

```javascript
process.stdout.write(event.delta);
```

Output becomes:

```text
Hello world this is a story...
```

which resembles ChatGPT's typing effect.

## Complete Streaming Example

```js
import OpenAI from 'openai';

const OPENAI_API_KEY = 'API-KEY';

const client = new OpenAI({
    apiKey: OPENAI_API_KEY,
});

const stream = await client.responses.create({
    model: 'gpt-3.5-turbo',
    input: 'Write a story about a robot',
    temperature: 0.7,
    max_output_tokens: 250,
    stream: true,
});

// stream is an async iterable.
// for await (const event of stream) console.log(event);
for await (const event of stream)
    if (event.delta) process.stdout.write(event.delta);
```
